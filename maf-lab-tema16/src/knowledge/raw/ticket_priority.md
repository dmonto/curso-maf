# Criterios de prioridad

Una incidencia P1 implica caída generalizada o impacto crítico en negocio.

Una incidencia P2 implica degradación importante o bloqueo de un grupo relevante de usuarios.

Una incidencia P3 implica impacto limitado a un usuario o workaround disponible.

Para problemas individuales de VPN, la prioridad habitual es P3 salvo que afecte a un perfil crítico.
